‚õ∞Ô∏è Heaps & Priority Queues

What is a Heap?

A heap is a specialized tree-based data structure that satisfies the heap property. It's commonly used to implement priority queues.

Heap Properties:
  • Complete Binary Tree: All levels filled except possibly the last, which fills left to right
  • Max Heap: Parent ‚â• Children (root is maximum)
  • Min Heap: Parent ‚â§ Children (root is minimum)
Min Heap Example: 
          1
        /   \
       3     2
      / \   / \
     5   4 7   6

Parent is always smaller than children!
Root (1) is the minimum element.
Max Heap Example: 
          10
        /    \
       8      9
      / \    / \
     4   5  6   7

Parent is always larger than children!
Root (10) is the maximum element.

Why Use Heaps?

  • Fast access to min/max: O(1) to peek at top element
  • Efficient insertion/deletion: O(log n) operations
  • Priority Queue: Process elements by priority, not order
  • Heap Sort: Sorting algorithm with O(n log n) time
  • Finding K largest/smallest: Very efficient pattern

Heap Operations & Complexity

Insert: O(log n) Extract Min/Max: O(log n) Peek Min/Max: O(1) Build Heap: O(n)

üìä Heap Insert & Extract Min Operations

üîº Insert: Add at end ‚Üí Bubble up to maintain heap property | üîΩ Extract Min: Remove root ‚Üí Move last to root ‚Üí Bubble down

Using Python's heapq Module

heapq - Python's Built-in Min Heap
import heapq

# Python's heapq implements a MIN HEAP

# Create a heap from a list
nums = [5, 2, 8, 1, 9]
heapq.heapify(nums)  # Converts list to heap in-place, O(n)
print(nums)  # [1, 2, 8, 5, 9] - heap structure

# Push (insert) element - O(log n)
heapq.heappush(nums, 3)
print(nums)  # [1, 2, 3, 5, 9, 8]

# Pop (extract min) - O(log n)
min_val = heapq.heappop(nums)
print(f"Minimum: {min_val}")  # 1
print(nums)  # [2, 5, 3, 8, 9]

# Peek at minimum without removing - O(1)
if nums:
    min_peek = nums[0]
    print(f"Peek minimum: {min_peek}")  # 2

# Get N smallest elements
nums = [5, 2, 8, 1, 9, 3]
smallest_3 = heapq.nsmallest(3, nums)
print(f"3 smallest: {smallest_3}")  # [1, 2, 3]

# Get N largest elements
largest_3 = heapq.nlargest(3, nums)
print(f"3 largest: {largest_3}")  # [9, 8, 5]
Creating a MAX HEAP in Python:

Python's heapq only supports min heap. To use it as max heap:

  • Method 1: Negate values when pushing, negate again when popping
  • Method 2: Use tuple (-priority, value) and heapq will sort by first element
Max Heap Using heapq (Negate Values)
import heapq

# Max heap by negating values
class MaxHeap:
    def __init__(self):
        self.heap = []
    
    def push(self, val):
        # Negate value for max heap behavior
        heapq.heappush(self.heap, -val)
    
    def pop(self):
        # Negate again when popping to get original value
        return -heapq.heappop(self.heap)
    
    def peek(self):
        return -self.heap[0] if self.heap else None
    
    def is_empty(self):
        return len(self.heap) == 0

# Usage:
max_heap = MaxHeap()
max_heap.push(5)
max_heap.push(2)
max_heap.push(8)
print(max_heap.pop())   # 8 (maximum)
print(max_heap.peek())  # 5 (new maximum)

Pattern 1: Find Kth Largest/Smallest Element

Kth Largest Element
import heapq

def find_kth_largest(nums, k):
    """
    Find kth largest element in array.
    
    Example: nums = [3,2,1,5,6,4], k = 2
    Output: 5 (second largest)
    
    Strategy: Use min heap of size k.
    - Keep only k largest elements in heap
    - Root of min heap will be kth largest!
    
    Time: O(n log k), Space: O(k)
    """
    # Maintain min heap of size k
    heap = []
    
    for num in nums:
        heapq.heappush(heap, num)
        
        # If heap exceeds size k, remove smallest
        if len(heap) > k:
            heapq.heappop(heap)
    
    # Root is kth largest
    return heap[0]

# Simpler approach using nlargest
def find_kth_largest_v2(nums, k):
    return heapq.nlargest(k, nums)[-1]

# Test:
nums = [3, 2, 1, 5, 6, 4]
print(find_kth_largest(nums, 2))  # 5
print(find_kth_largest(nums, 4))  # 3

Pattern 2: Merge K Sorted Lists

Merge K Sorted Lists Using Heap
import heapq

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def merge_k_lists(lists):
    """
    Merge k sorted linked lists.
    
    Example:
    Input: [[1,4,5], [1,3,4], [2,6]]
    Output: [1,1,2,3,4,4,5,6]
    
    Strategy: Use min heap to always get smallest element.
    - Add first node from each list to heap
    - Pop smallest, add its next to heap
    
    Time: O(N log k) where N is total nodes, k is number of lists
    Space: O(k) for heap
    """
    heap = []
    
    # Add first node from each list
    for i, node in enumerate(lists):
        if node:
            # (value, index, node) - index ensures stable ordering
            heapq.heappush(heap, (node.val, i, node))
    
    # Create dummy head
    dummy = ListNode(0)
    current = dummy
    
    while heap:
        # Get smallest node
        val, i, node = heapq.heappop(heap)
        
        # Add to result
        current.next = node
        current = current.next
        
        # Add next node from same list
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
    
    return dummy.next

Pattern 3: Top K Frequent Elements

Find K Most Frequent Elements
import heapq
from collections import Counter

def top_k_frequent(nums, k):
    """
    Find k most frequent elements.
    
    Example: nums = [1,1,1,2,2,3], k = 2
    Output: [1, 2]
    
    Time: O(n log k), Space: O(n)
    """
    # Count frequencies
    count = Counter(nums)
    
    # Use min heap of size k
    heap = []
    
    for num, freq in count.items():
        heapq.heappush(heap, (freq, num))
        
        if len(heap) > k:
            heapq.heappop(heap)
    
    # Extract elements
    return [num for freq, num in heap]

# Alternative using nlargest
def top_k_frequent_v2(nums, k):
    count = Counter(nums)
    return heapq.nlargest(k, count.keys(), key=count.get)

# Test:
nums = [1,1,1,2,2,3]
print(top_k_frequent(nums, 2))  # [1, 2]

Pattern 4: Median from Data Stream

Find Median Using Two Heaps
import heapq

class MedianFinder:
    """
    Find median from data stream.
    
    Strategy: Use TWO heaps!
    - Max heap for smaller half (left side)
    - Min heap for larger half (right side)
    - Balance them so difference in size ‚â§ 1
    
    Median is:
    - If sizes equal: average of both tops
    - If one larger: top of larger heap
    
    Time: O(log n) for addNum, O(1) for findMedian
    """
    
    def __init__(self):
        # Max heap for smaller half (negate for max behavior)
        self.small = []
        # Min heap for larger half
        self.large = []
    
    def addNum(self, num):
        # Add to max heap (small half)
        heapq.heappush(self.small, -num)
        
        # Move largest of small to large
        heapq.heappush(self.large, -heapq.heappop(self.small))
        
        # Balance heaps (small can have at most 1 more element)
        if len(self.small) < len(self.large):
            heapq.heappush(self.small, -heapq.heappop(self.large))
    
    def findMedian(self):
        if len(self.small) > len(self.large):
            return -self.small[0]
        else:
            return (-self.small[0] + self.large[0]) / 2

# Test:
mf = MedianFinder()
mf.addNum(1)
mf.addNum(2)
print(mf.findMedian())  # 1.5
mf.addNum(3)
print(mf.findMedian())  # 2

What is a Graph?

A graph is a data structure consisting of vertices (nodes) connected by edges (links). Graphs are used to represent networks, relationships, and connections.

Real-world examples:

  • Social networks (users = nodes, friendships = edges)
  • Road maps (cities = nodes, roads = edges)
  • Computer networks (computers = nodes, connections = edges)
  • Web pages (pages = nodes, hyperlinks = edges)
  • Flight routes, course prerequisites, dependency graphs
Graph Terminology:
  • Vertex/Node: A point in the graph
  • Edge: Connection between two vertices
  • Adjacent: Two vertices connected by an edge
  • Degree: Number of edges connected to a vertex
  • Path: Sequence of vertices connected by edges
  • Cycle: Path that starts and ends at same vertex
  • Connected: Path exists between any two vertices

Types of Graphs

Undirected Graph (edges have no direction): 
    A --- B
    |     |
    C --- D

A-B, A-C, B-D, C-D are bidirectional edges
Directed Graph (edges have direction): 
    A --> B
    ‚Üì     ‚Üì
    C --> D

A‚ÜíB, A‚ÜíC, B‚ÜíD, C‚ÜíD are one-way edges
Weighted Graph (edges have values): 
    A --5-- B
    |       |
    3       2
    |       |
    C --4-- D

Edge weights: A-B=5, A-C=3, B-D=2, C-D=4

Graph Traversal - THE Most Important Graph Algorithms!

1. Breadth-First Search (BFS)

Explore level by level - like ripples in water. Uses QUEUE.

BFS Traversal Order: 
        A
       / \
      B   C
     /   / \
    D   E   F

Starting from A:
Level 0: A
Level 1: B, C
Level 2: D, E, F

BFS Order: A ‚Üí B ‚Üí C ‚Üí D ‚Üí E ‚Üí F
BFS Implementation
from collections import deque

def bfs(graph, start):
    """
    Breadth-First Search traversal.
    
    Uses: Shortest path, level-order, nearest neighbor
    
    Time: O(V + E) where V = vertices, E = edges
    Space: O(V) for queue and visited set
    """
    visited = set()
    queue = deque([start])
    visited.add(start)
    result = []
    
    while queue:
        vertex = queue.popleft()
        result.append(vertex)
        
        # Visit all neighbors
        for neighbor in graph[vertex]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    
    return result

# Example graph
graph = {
    'A': ['B', 'C'],
    'B': ['A', 'D'],
    'C': ['A', 'E', 'F'],
    'D': ['B'],
    'E': ['C'],
    'F': ['C']
}

print(bfs(graph, 'A'))  # ['A', 'B', 'C', 'D', 'E', 'F']

2. Depth-First Search (DFS)

Explore as deep as possible before backtracking. Uses STACK (or recursion).

DFS Traversal Order: 
        A
       / \
      B   C
     /   / \
    D   E   F

Starting from A, going left first:
A ‚Üí B ‚Üí D (dead end, backtrack)
  ‚Üí C ‚Üí E (backtrack)
      ‚Üí F

DFS Order: A ‚Üí B ‚Üí D ‚Üí C ‚Üí E ‚Üí F
DFS Implementation (Recursive & Iterative) 
# DFS - Recursive (Most Common)
def dfs_recursive(graph, vertex, visited=None):
    """
    Depth-First Search (recursive).
    
    Uses: Detect cycles, topological sort, path finding
    
    Time: O(V + E)
    Space: O(V) for recursion stack
    """
    if visited is None:
        visited = set()
    
    visited.add(vertex)
    result = [vertex]
    
    for neighbor in graph[vertex]:
        if neighbor not in visited:
            result.extend(dfs_recursive(graph, neighbor, visited))
    
    return result

# DFS - Iterative (using stack)
def dfs_iterative(graph, start):
    """DFS using explicit stack (iterative)"""
    visited = set()
    stack = [start]
    result = []
    
    while stack:
        vertex = stack.pop()
        
        if vertex not in visited:
            visited.add(vertex)
            result.append(vertex)
            
            # Add neighbors in reverse to match recursive order
            for neighbor in reversed(graph[vertex]):
                if neighbor not in visited:
                    stack.append(neighbor)
    
    return result

# Test both
graph = {

    'B': ['A', 'D'],
    'C': ['A', 'E', 'F'],
    'D': ['B'],
    'E': ['C'],
    'F': ['C']
}

print(dfs_recursive(graph, 'A'))  # ['A', 'B', 'D', 'C', 'E', 'F']
print(dfs_iterative(graph, 'A'))  # ['A', 'B', 'D', 'C', 'E', 'F']

Find if Path Exists Between Two Nodes
def has_path(graph, start, end):
    """
    Check if path exists from start to end.
    
    Can use either BFS or DFS - both work!
    """
    if start == end:
        return True
    
    visited = set()
    queue = deque([start])
    visited.add(start)
    
    while queue:
        vertex = queue.popleft()
        
        if vertex == end:
            return True
        
        for neighbor in graph[vertex]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    
    return False

2. Number of Connected Components

Count Connected Components
def count_components(n, edges):
    """
    Count number of connected components in undirected graph.
    
    Example: n = 5, edges = [[0,1], [1,2], [3,4]]
    Output: 2 (components: {0,1,2} and {3,4})
    
    Time: O(V + E), Space: O(V)
    """
    # Build adjacency list
    graph = defaultdict(list)
    for a, b in edges:
        graph[a].append(b)
        graph[b].append(a)
    
    visited = set()
    count = 0
    
    def dfs(node):
        visited.add(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                dfs(neighbor)
    
    # Check each node
    for i in range(n):
        if i not in visited:
            dfs(i)
            count += 1
    
    return count

3. Shortest Path in Unweighted Graph

Find Shortest Path Using BFS
from collections import deque

def shortest_path(graph, start, end):
    """
    Find shortest path in unweighted graph.
    
    BFS ALWAYS gives shortest path in unweighted graphs!
    
    Returns: Path as list, or None if no path exists
    Time: O(V + E), Space: O(V)
    """
    if start == end:
        return [start]
    
    visited = {start}
    queue = deque([(start, [start])])  # (vertex, path_to_vertex)
    
    while queue:
        vertex, path = queue.popleft()
        
        for neighbor in graph[vertex]:
            if neighbor == end:
                return path + [neighbor]
            
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, path + [neighbor]))
    
    return None  # No path exists

    'B': ['A', 'D'],
    'C': ['A', 'D'],
    'D': ['B', 'C', 'E'],
    'E': ['D']
}

Detect Cycle in Undirected Graph
def has_cycle_undirected(graph):
    """
    Detect cycle in undirected graph using DFS.
    
    Strategy: During DFS, if we visit a node that's already
    visited AND it's not the parent, we found a cycle!
    
    Time: O(V + E), Space: O(V)
    """
    visited = set()
    
    def dfs(node, parent):
        visited.add(node)
        
        for neighbor in graph[node]:
            if neighbor not in visited:
                if dfs(neighbor, node):
                    return True
            elif neighbor != parent:
                # Visited node that's not parent = cycle!
                return True
        
        return False
    
    # Check all components
    for node in graph:
        if node not in visited:
            if dfs(node, None):
                return True
    
    return False

# Test with cycle

    0: [1, 2],
    1: [0, 2],
    2: [0, 1]
}
print(has_cycle_undirected(graph_with_cycle))  # True

# Test without cycle (tree)

    0: [1, 2],
    1: [0],
    2: [0]
}
print(has_cycle_undirected(graph_no_cycle))  # False

1. Dijkstra's Algorithm - Shortest Path in Weighted Graphs

Dijkstra's algorithm finds the shortest path from a source to all other vertices in a weighted graph with non-negative weights.

üí° Key Insight: Dijkstra uses a min-heap to always process the vertex with the smallest known distance. It's a greedy algorithm that guarantees the shortest path for non-negative weights.
Python Implementation - Dijkstra's Algorithm 
import heapq

def dijkstra(graph, start):
    # Initialize distances with infinity for all vertices
    distances = {vertex: float('inf') for vertex in graph}
    distances[start] = 0
    
    # Min-heap: (distance, vertex)
    heap = [(0, start)]
    visited = set()
    
    while heap:
        current_distance, current_vertex = heapq.heappop(heap)
        
        # Skip if already visited
        if current_vertex in visited:
            continue
        
        visited.add(current_vertex)
        
        # Check all neighbors
        for neighbor, weight in graph[current_vertex]:
            distance = current_distance + weight
            
            # Only update if we found a shorter path
            if distance < distances[neighbor]:
                distances[neighbor] = distance
                heapq.heappush(heap, (distance, neighbor))
    
    return distances

# Example usage
graph = {
    'A': [('B', 4), ('C', 1)],
    'B': [('D', 1)],
    'C': [('B', 2), ('D', 5)],
    'D': []
}

distances = dijkstra(graph, 'A')
print(f"Shortest distances from A: {distances}")
# Output: {'A': 0, 'B': 3, 'C': 1, 'D': 4}
Time: O((V + E) log V) Space: O(V)
‚ö†Ô∏è Important Limitation: Dijkstra's algorithm does NOT work with negative edge weights! For graphs with negative weights, use Bellman-Ford algorithm instead.

2. Bellman-Ford Algorithm - Handles Negative Weights

Bellman-Ford can handle negative edge weights and detect negative cycles. It's slower than Dijkstra but more versatile.

Python Implementation - Bellman-Ford Algorithm 
def bellman_ford(graph, start, num_vertices):
    # Initialize distances
    distances = {i: float('inf') for i in range(num_vertices)}
    distances[start] = 0
    
    # Relax all edges (V-1) times
    for _ in range(num_vertices - 1):
        for u in graph:
            for v, weight in graph[u]:
                if distances[u] + weight < distances[v]:
                    distances[v] = distances[u] + weight
    
    # Check for negative cycles (one more iteration)
    for u in graph:
        for v, weight in graph[u]:
            if distances[u] + weight < distances[v]:
                return "Graph contains negative cycle"
    
    return distances

# Example with negative weight
graph = {
    0: [(1, 1)],           # A ‚Üí B (weight 1)
    1: [(2, -3)],          # B ‚Üí C (weight -3)
    2: [(3, 2)],           # C ‚Üí D (weight 2)
    3: []
}

result = bellman_ford(graph, 0, 4)
print(f"Shortest distances: {result}")
Time: O(V √ó E) Space: O(V)

3. Topological Sort - Ordering with Dependencies

Topological sort orders vertices in a Directed Acyclic Graph (DAG) such that for every edge u‚Üív, u comes before v in the ordering.

üéØ Real-World Example - Course Prerequisites:

If Course B requires Course A as a prerequisite, then A must come before B in the schedule. Topological sort finds a valid course completion order.

Method: Kahn's Algorithm (BFS with In-Degree)

Python - Kahn's Algorithm (BFS)
from collections import deque

def topological_sort_kahns(graph, num_vertices):
    # Calculate in-degree for each vertex
    in_degree = {i: 0 for i in range(num_vertices)}
    for vertex in graph:
        for neighbor in graph[vertex]:
            in_degree[neighbor] += 1
    
    # Start with vertices that have no prerequisites
    queue = deque([v for v in in_degree if in_degree[v] == 0])
    result = []
    
    while queue:
        vertex = queue.popleft()
        result.append(vertex)
        
        # Remove this vertex's edges
        for neighbor in graph[vertex]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)
    
    # If all vertices processed, return result; else cycle detected
    return result if len(result) == num_vertices else None

# Example usage
courses = {
    0: [2],  # Course 0 prerequisite for Course 2
    1: [2],  # Course 1 prerequisite for Course 2
    2: [3],  # Course 2 prerequisite for Course 3
    3: []
}

order = topological_sort_kahns(courses, 4)
print(f"Course completion order: {order}")
# Output: [0, 1, 2, 3] or [1, 0, 2, 3]
Time: O(V + E) Space: O(V)

4. Minimum Spanning Tree (MST)

A Minimum Spanning Tree connects all vertices with the minimum total edge weight, forming a tree (no cycles).

Union-Find (Disjoint Set): A data structure that efficiently tracks which vertices are in the same connected component. It supports two operations:
  • Find: Determine which set a vertex belongs to
  • Union: Merge two sets together
Python - Union-Find Implementation
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
    
    def find(self, x):
        # Path compression: make x point directly to root
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]
    
    def union(self, x, y):
        # Union by rank: attach smaller tree to larger
        root_x = self.find(x)
        root_y = self.find(y)
        
        if root_x == root_y:
            return False  # Already in same set
        
        if self.rank[root_x] < self.rank[root_y]:
            self.parent[root_x] = root_y
        elif self.rank[root_x] > self.rank[root_y]:
            self.parent[root_y] = root_x
        else:
            self.parent[root_y] = root_x
            self.rank[root_x] += 1
        
        return True  # Successfully merged
Python - Kruskal's Algorithm with Union-Find 
def kruskals_mst(num_vertices, edges):
    # Sort edges by weight
    edges.sort(key=lambda x: x[2])
    
    uf = UnionFind(num_vertices)
    mst = []
    total_weight = 0
    
    for u, v, weight in edges:
        # Add edge if it doesn't create a cycle
        if uf.union(u, v):
            mst.append((u, v, weight))
            total_weight += weight
            
            # Stop when we have V-1 edges
            if len(mst) == num_vertices - 1:
                break
    
    return mst, total_weight

# Example usage
edges = [
    (0, 1, 1),  # A-B weight 1
    (0, 2, 2),  # A-C weight 2
    (0, 3, 3),  # A-D weight 3
    (1, 3, 4),  # B-D weight 4
    (2, 3, 5),  # C-D weight 5
]

mst, weight = kruskals_mst(4, edges)
print(f"MST edges: {mst}")
print(f"Total weight: {weight}")
# Output: MST edges: [(0, 1, 1), (0, 2, 2), (1, 3, 4)], Total: 7
Time: O(E log E) Space: O(V)

5. Floyd-Warshall - All-Pairs Shortest Path

Floyd-Warshall finds shortest paths between all pairs of vertices. It works with negative weights but not negative cycles.

Python - Floyd-Warshall Algorithm
def floyd_warshall(graph, num_vertices):
    # Initialize distance matrix
    dist = [[float('inf')] * num_vertices for _ in range(num_vertices)]
    
    # Distance from vertex to itself is 0
    for i in range(num_vertices):
        dist[i][i] = 0
    
    # Fill in direct edges
    for u in graph:
        for v, weight in graph[u]:
            dist[u][v] = weight
    
    # Try each vertex as intermediate point
    for k in range(num_vertices):
        for i in range(num_vertices):
            for j in range(num_vertices):
                dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
    
    return dist

# Example usage
graph = {
    0: [(1, 1), (3, 5)],  # A ‚Üí B(1), A ‚Üí D(5)
    1: [(3, 2)],           # B ‚Üí D(2)
    2: [(3, 6)],           # C ‚Üí D(6)
    3: []
}

distances = floyd_warshall(graph, 4)
print("All-pairs shortest distances:")
for i, row in enumerate(distances):
    print(f"From vertex {i}: {row}")
Time: O(V¬≥) Space: O(V¬≤)
üí° When to Use Floyd-Warshall: Use when you need distances between ALL pairs of vertices. If you only need single-source shortest paths, Dijkstra or Bellman-Ford are more efficient.
